1. Abstract

This project presents a comprehensive big data pipeline for analyzing and predicting apartment prices in the Egyptian real estate market. We scraped over 70,000 apartment listings from Dubizzle Egypt, applied both regex-based and Large Language Model (LLM)–assisted feature extraction, performed extensive data cleaning and exploratory analysis, and trained a LightGBM gradient boosting model that predicts apartment prices from 49 engineered features. The entire system is deployed as a FastAPI REST backend connected to an interactive Streamlit dashboard that enables end users to explore market trends, search and filter listings, compare properties side-by-side, and obtain instant ML-powered price predictions.

2. Problem Description

The Egyptian real estate market is characterized by significant price opacity and information asymmetry. Apartment prices vary dramatically based on location, amenities, finish level, and compound affiliation, yet buyers have limited tools to objectively evaluate whether a listed price represents fair market value. Key problems include:

  • Price Opacity: Sellers set prices subjectively with no standardized valuation framework, leading to frequent overpricing and occasional underpricing.
  • Information Fragmentation: Listing data is scattered across platforms with inconsistent formatting, missing fields, and unstructured descriptions.
  • Lack of Analytical Tools: No existing platform provides Egyptian buyers with data-driven insights such as price-per-sqm benchmarks, neighborhood comparisons, or ML-based fair-price estimates.
  • Scam Detection Gap: The absence of automated price validation allows misleading or fraudulent listings to persist.

3. Project Objectives

  1. Build a large-scale web scraper to collect nationwide apartment listing data from Dubizzle Egypt.
  2. Apply hybrid feature extraction (regex + Gemini LLM) to parse unstructured listing text into structured features.
  3. Perform thorough data cleaning, outlier removal, and exploratory data analysis.
  4. Engineer domain-specific features (interaction ratios, amenity scores, delivery buckets) and train a state-of-the-art gradient boosting model.
  5. Deploy the model as a production-ready REST API using FastAPI.
  6. Build an interactive dashboard that empowers users to explore the market, search listings, compare properties, and predict fair prices.

6. Methodology & Technical Description

6.1 Web Scraping

We developed a Python-based scraper using Selenium and BeautifulSoup to systematically crawl Dubizzle Egypt's apartment-for-sale category. The scraper was designed with:

  • Pagination handling to traverse all result pages across every Egyptian governorate.
  • Incremental saving with CSV checkpointing to resume from interruptions.
  • Rate limiting and randomized delays to respect the platform's usage policies.
  • Error recovery with exponential backoff for transient network failures.

The scraper collected raw HTML text for each listing including title, price string, location text, description, and attribute tables.

6.2 Feature Extraction (Regex + LLM)

Raw listing data required extensive parsing to extract structured features. We employed a two-stage hybrid approach:

  1. Regex-Based Extraction: Pattern matching for well-formatted fields (price, area in sqm, number of rooms/bathrooms) using regular expressions.
  2. LLM-Assisted Extraction: For ambiguous or unstructured fields (amenities, finish type, compound name, view type), we used the Google Gemini API to parse free-text descriptions into structured JSON. This approach achieved significantly higher coverage than regex alone, particularly for Arabic-language listings.

6.3 Data Cleaning & Visualization

Cleaning steps included:

  • Removal of duplicate listings (by URL hash).
  • Type coercion and null imputation for numeric fields.
  • Outlier detection using IQR and z-score methods on price and area.
  • Standardization of city/governorate names (lowercasing, Arabic transliteration).
  • Validation of latitude/longitude against known Egyptian boundaries.

Extensive exploratory data analysis was performed in the Dubizzle_Apartments_Enhanced_SOTA.ipynb notebook, including distribution plots, correlation heatmaps, geographic scatter maps, and cross-tabulations.

6.5 Model Training & Evaluation

We trained the model on log1p(price) to handle the right-skewed price distribution. The preprocessing pipeline consists of:

  1. Target Encoding (via category_encoders.TargetEncoder) for high-cardinality categorical features (city, seller_name, compound_name, etc.).
  2. Standard Scaling (via sklearn.StandardScaler) for all numeric features.
  3. LightGBM Gradient Boosting as the final estimator.
Model: LightGBM (lgb_baseline.txt) trained on 60,280 samples with 48 features. Predictions are inverse-transformed using numpy.expm1() to return prices in EGP.

6.6 Backend Deployment (FastAPI)

The trained model is served via a FastAPI REST API (main.py) with the following endpoints:

MethodEndpointDescription
GET/api/explore/statsAggregated market statistics from the full 70K dataset
GET/api/explore/dataDataset rows as JSON with optional governorate filter
GET/api/searchFull-filter search over SQLite-backed test data
GET/api/search/filtersAvailable filter values for dropdowns
GET/api/compareMulti-apartment comparison by IDs
POST/models/lgb_baselineReal-time price prediction

The test dataset (test_data.csv, 10,638 rows) is automatically loaded into a SQLite database on first server start, with indexes on frequently filtered columns (city, governorate, property_type, bedrooms, price) for sub-millisecond query performance.

6.7 Interactive Dashboard (Streamlit)

The front-end dashboard (app.py) provides four interactive tabs:

  1. Explore Market: Heatmap of real prices on a map, interactive charts for price insights by city, governorate, property type, and amenities.
  2. Search: Multi-filter apartment search with cascading governorate → city dropdowns, per-listing ML prediction with deal-verdict badges (Fair Price / Great Deal / Overpriced / Possibly a Scam).
  3. Compare: Side-by-side feature table, radar chart, and deal-ranking list sorted by predicted-vs-actual price delta.
  4. Price Predictor: Interactive form where users input property specifications and receive instant LightGBM price estimates. City coordinates are auto-filled from a median lat/lng lookup table.

7. System Architecture

The system follows a three-tier architecture:

TierComponentTechnology
Data LayerRaw CSV data + SQLite databasePandas, SQLite3
Application LayerFastAPI REST backend with ML inferenceFastAPI, LightGBM, scikit-learn
Presentation LayerStreamlit interactive dashboardStreamlit, Plotly, Folium

The data flows as follows:

  1. Scraping → CSV: Raw listings scraped from Dubizzle and stored as CSV files.
  2. CSV → Notebook: Data cleaned, features engineered, and model trained in Jupyter.
  3. Models → FastAPI: Saved model artifacts loaded at server startup.
  4. CSV → SQLite: Test data auto-loaded into SQLite for fast querying.
  5. FastAPI ↔ Streamlit: Dashboard calls REST endpoints for all data and predictions.
